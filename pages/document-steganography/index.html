文档隐写实验室

两种商业级文档追踪技术的真实演示。
PDF 追踪像素：文件一被打开，服务器就知道是谁、在哪里、用什么设备。
文字指纹水印：给不同人发不同版本，文件泄露后一查便知是谁干的。

生成追踪文档

生成一份看起来普通的 PDF。里面嵌入了一个 1×1 的透明追踪像素， 指向我们的服务器。收件方一打开，服务器就会收到请求。

追踪 ID —
把这份 PDF 发给任何人，或者自己用 PDF 阅读器打开，右侧就会实时出现记录。

实时打开记录

每隔 3 秒自动刷新。IP 已打码，仅显示前两段。

等待文件被打开…
共 0 次打开记录

这是怎么做到的

1
嵌入远程图片

PDF 格式支持引用外部 URL 的图片。我们在文档里放一个 1×1 透明 PNG，地址指向追踪服务器，附带唯一 token。

2
阅读器自动请求

Adobe Acrobat、Foxit、浏览器内置 PDF 阅读器在渲染页面时会自动加载图片资源，触发 HTTP 请求。

3
服务器记录信息

服务器收到请求，记录 IP 地址、User-Agent（设备和系统信息）、打开时间，与 token 关联存储。

4
实时回传

发件方的页面每 3 秒轮询一次，有新记录就立刻显示。你知道文件被打开了，对方完全不知道。

注意：部分 PDF 阅读器（如 iOS 自带预览）会拦截外部请求，或需要用户手动允许加载远程内容。企业级方案通常会把追踪像素伪装成文档 logo 或页眉图片，降低被拦截概率。

注入文字指纹

输入一段文本，为不同收件方生成看起来完全一样、但指纹不同的版本。 文件泄露后，把泄露版本粘贴进来，就能知道是谁泄露的。

指纹提取 / 溯源

把你收到的「疑似泄露版本」粘贴进来，系统会提取指纹并告诉你是哪个收件方泄露的。

文字指纹是怎么工作的

1
同义词替换

「总金额」↔「总价款」，「交付」↔「完成」，「保证」↔「确保」。每个替换点是一个 bit，0 或 1。

2
标点变体

全角逗号「，」↔ 全角顿号「、」，句号「。」后加零宽空格。肉眼看不出差异，但字节不同。

3
编码为 ID

N 个替换点可以编码 2ᴺ 个不同版本。10 个替换点 = 1024 个不同收件方，每人一份唯一版本。

4
提取溯源

拿到泄露版本，逐个检查替换点的选择，还原出 ID，对应到具体收件方。

局限性：如果泄露方对文本做了大幅改写或翻译，指纹可能失效。商业级方案（如 Digimarc）会结合字间距微调、行高变化等更隐蔽的方式，抗改写能力更强。
