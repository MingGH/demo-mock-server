这个 API 到底解决了什么

01

渲染引擎一致性

html2canvas 在 JS 里"猜"浏览器怎么渲染，复杂 CSS 一定会翻车。drawElementImage 直接喂给 Blink，输出就是浏览器自己画的。

02

Emoji / RTL / 复杂排版

fillText 不会处理双向文本和复杂脚本。新 API 直接复用浏览器的排版引擎，阿拉伯语、印地语、彩色 emoji 一次到位。

03

backdrop-filter / mix-blend

这些靠 GPU 合成的效果在 html2canvas 里直接被忽略。新 API 走的就是合成层，所见即所得。

04

3D / WebGL 纹理

配套的 texElementImage2D 让 WebGL 把 DOM 当纹理用——3D 场景里贴一张可交互网页，不再需要 hack。

三行 API 看懂它怎么工作

// 1. 给 canvas 加 layoutsubtree，让子 DOM 走真正的布局流程
<canvas layoutsubtree width="340" height="200">
  <div class="poster">...真正的 HTML/CSS...</div>
</canvas>

// 2. paint 事件：DOM 变化时浏览器主动喊你重绘
canvas.onpaint = () => redraw();

// 3. 把 DOM 节点直接画进 Canvas 2D 上下文
const ctx = canvas.getContext('2d');
ctx.drawElementImage(poster, 0, 0);

// WebGL 同款：DOM 作为 3D 纹理
gl.texElementImage2D(gl.TEXTURE_2D, 0, gl.RGBA, gl.RGBA, gl.UNSIGNED_BYTE, poster);

把一张活的 DOM 卡片，炸成像素

DOM 卡片可编辑（试着改一下标题或按钮）。点"炸"之后整张卡片被采样成几千个粒子做物理模拟。这就是 HTML-in-Canvas 想给开发者的新自由度：DOM 不再是 DOM，是一份可任意操作的像素源。
