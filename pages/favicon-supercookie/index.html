Favicon 超级 Cookie

你清了 Cookie、开了无痕、挂了 VPN，网站照样认出你。
因为你浏览器里有个缓存，你根本不知道它存在。

什么是 Favicon？

浏览器标签页上那个小图标，就是 favicon。每次你访问一个网站，浏览器会把这个图标缓存到一个独立的数据库里——F-Cache。

关键来了：这个 F-Cache 跟普通缓存不在一起。你点「清除浏览数据」的时候，Cookie 没了、LocalStorage 没了、浏览器缓存也没了——但 F-Cache 还在。

2021 年，伊利诺伊大学芝加哥分校的研究者发表了论文 「Tales of Favicons and Caches: Persistent Tracking in Modern Browsers」（NDSS 2021），证明了利用 F-Cache 可以在 2 秒内重建一个 32 位的追踪 ID。

追踪原理：用「有没有缓存」编码 0 和 1

核心思路

网站准备 N 个子域名（bit0.tracker.com、bit1.tracker.com ...），每个子域名有自己的 favicon。

给你分配 ID 时，让你的浏览器访问其中一部分子域名（对应二进制位 = 1），跳过另一部分（= 0）。

下次你再来，网站逐个检查：浏览器有没有请求这个子域名的 favicon？
请求了 → 没缓存 → 0，没请求 → 有缓存 → 1。

N 个子域名 = N 位二进制 = 可以标记 2^N 个用户。32 个子域名就够标记 40 亿人。

怎么防？

手动清除 F-Cache

浏览器没有提供「清除 Favicon 缓存」的按钮，你得手动删文件：

Chrome (macOS):
~/Library/Application Support/Google/Chrome/Default/Favicons

Safari (macOS):
~/Library/Safari/Favicon Cache/

Chrome (Windows):
C:\Users\用户名\AppData\Local\Google\Chrome\User Data\Default\Favicons

换用已修复的浏览器

Brave 浏览器从 v1.19 开始已修复此漏洞。Firefox 84+ 在无痕模式下使用独立的 F-Cache。
但在普通模式下，大多数浏览器仍然存在这个问题。

已知的滥用场景

研究者在论文中指出，这种追踪技术可以被广告网络、数据经纪商用于跨站追踪。由于不依赖 Cookie，它完全绕过了 GDPR 的 Cookie 同意弹窗。

更关键的是：用户完全无法感知这个过程。没有弹窗、没有提示、没有权限请求。浏览器在后台默默完成了一切。

Chromium 团队在 2020 年收到了相关的安全报告（Issue #1096660），但截至目前 Chrome 仍未完全修复。
