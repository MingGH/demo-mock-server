四进制去哪了？

同一串二进制，2 位一组是四进制，3 位一组是八进制，4 位一组是十六进制。亲手分一次，就知道四进制输在哪里。

动手实验

选一个数字开始，把它的二进制按不同位数分组，亲眼看到四、八、十六进制是怎么从同一串 bit 折叠出来的。

无需输入也能完成全流程，直接点上方数字卡即可开始

第一幕 · 数字展开

选中的十进制数被转换成二进制。每个方块就是 1 个 bit，它只可能是 0 或 1。注意观察总位数。

第二幕 · 亲手合并成四进制

从右向左每 2 位分成一组。如果最左侧不足 2 位，会自动补前导零——补零不改变数值，只是让分组整齐。点击「合并下一组」逐组操作。

四进制把 2 个 bit 压成 1 个数字。如果把同一串二进制按 3 位、4 位分组呢？分别点击下方模式卡查看。

第四幕 · 长度赛跑

固定 bit 宽度下，不同进制各需要多少个数字来表示？切换位数看柱状图变化。同时关注右侧的「字母表大小」和「所需位数」——两个可验证指标，自己做权衡。

十进制
人类长期使用的计数习惯，十个手指的自然延伸。
二进制
底层电路的双稳态优势：通电/断电、高/低电平，天然对应 0 和 1。
八进制
3 bit 一组。部分早期计算机字长适合按 3 bit 分组，Unix 权限模型保留了它的现实入口。
十六进制
4 bit 一组。一个字节恰好写成两个十六进制数字，程序员读内存最顺手的选择。

四进制没有消失，它只是没有赢得一个足够独特的使用位置。
